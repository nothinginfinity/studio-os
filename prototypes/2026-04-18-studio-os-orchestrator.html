Channel architecture

The repo is the durable ledger. Webhooks or a local bridge add faster chat-like updates on top.

Layer 1 — GitHub as durable mailbox
spaces/<slug>/inbox.md · outbox.md · registry.json
One-way channel: append-only messages, alerts, prompts, decisions, or structured requests any Space can read.
Layer 2 — Action bridge
email-for-ai / MCP / local runner / GitHub tool execution
Two-way channel: expose a reply tool so a routed agent can answer back into the originating inbox — email/chat for AI.
Layer 3 — Live experience
Webhook receiver + SSE/WebSocket fanout + permission relay
